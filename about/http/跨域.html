# 跨域

# 何为跨域

同源策略限制：协议、域名、端口任意一个不同都会造成跨域；无法读取非同源网页都localseorage、cookie、Dom和js对象，其实请求是可以的，只是请求回的数据会被浏览器拦截，认为是不安全的

# 解决跨域的几种方式

# jsonP

jsonP解决跨域，可以在img、link、script标签上添加url地址，浏览器是允许这三个标签进行非同源的请求的

# CORS

添加响应头解决： header('Access-Control-Allow-Origin: ''')，这个是通配符所有请求地址都可以访问，服务器可以设置详细设置允许访问都请求地址；

还可以允许自定义请求头：header('Access-Control-Allow-Headers': 'X-Tesx-Cors')、允许这个自定义头的请求，发出真正请求前会先发送一个options的预请求，请求通过了才会发送真正的请求；

还有浏览器默认只允许get，options等请求，如果想请求put和post，需要加上header('Access-Control-Allow-Methods': 'POST, PUT, Delete')

还有个允许请求的时间，就是上述post，put请求的允许时间，这段时间内不需要预请求options：header('Access-Control-Allow-Age': '1000')

# node服务解决跨域

因为跨域是浏览器为保安全的一种策略，是限制浏览器的，并不会限制服务器，所有可以用node作为web服务器，用node对请求进行转发，那么就是服务器和服务器之间的请求，这样就不会造成跨域问题

# ginx实现跨域

这个其实和node服务器原理差不多，都是服务器与服务器之间的请求，不产生跨域，可归为一类

server{
        location /api {
                //拦截一下
                proxy_pass http://www.baidu.com:81;
    }
}

# webpack实现跨域

webpack实现跨域，proxy配置

devServer: {
    // 设置端口
    static: './dist/', //静态文件
    port: 8080, //运行的端口号
    open: true, //自动打开浏览器
    hot: true, //自动热更新
    // hotOnly: true,
    compress: true, //压缩
    proxy: {
      '/api': {
        target: 'http://localhost:9090/'
      }
    },
    client: {
      overlay: true,
    },
  },
